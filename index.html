💡 功能特色與評級機制說明

🚀 核心功能特色

  • 即時連線探測：每秒探測一次，支援 P50/P95 百分位圖表與雙端點 Failover 自動切換。
  • 抖動與尖峰記錄：滾動計算網路 Jitter 與 MAD，自動擷取異常脈衝尖峰。
  • 失敗／逾時可靠度：結合 Wilson 信心區間，精準反映真實丟包與 Timeout。
  • 壅塞模擬測試：並行大流量背景下載，評估網路在負載下的 Bufferbloat 衝擊。
  • 分頁省電保護：切換至背景分頁時自動暫停繪製，防止 GPU/CPU 資源浪費。

📊 企業評級機制標準 (0 - 100 分)

評級綜合考量延遲 (35%)、抖動 (25%)、遺失/逾時 (30%) 與尖峰 (10%)。門檻放寬以貼合辦公室與倉庫 PDA 條碼重傳 API 之需求：

評級 分數 實用品質體驗與建議
S+ / S 90 - 100 極佳：超低延遲 (<50ms)，無任何連線中斷。
A+ / A 75 - 89 良好：延遲 180ms~450ms 內，適合辦公室日常、線上會議與倉庫 PDA 掃碼列印。
B 60 - 74 勉強：偶有延遲波動，高頻即時操作可能微有感知。
C / F < 60 不穩定／差：高延遲或頻繁 Timeout，請檢查網路連線。

💡 動態解鎖門檻：當樣本數 (N) > 100 時，需累積採樣達 60 次才解鎖顯示評級；當 N ≤ 100 時則需達 30 次，防範初期數據過少造成評級失真。

📐 具體評級計算公式 (Exact Calculation Formula)

  • 加權總分算式：
    Score = 35%×延遲 + 25%×抖動 + 30%×遺失率 + 10%×尖峰率
  • 短板瓶頸封頂防護 (Bottleneck Cap)：
    FinalScore = min( 加權總分, 遺失率分數 + 25, 延遲分數 + 25 ) 當連線嚴重斷線時（遺失率 0分），總分將被硬性鎖定在最高 25 分 (F 級)，防範木桶缺陷。
  • 各指標滿分/零分區間門檻：
    • 延遲 (p50/p95): 220~600ms / 400~900ms   • 抖動: 60~180ms   • 遺失率: 4~16%   • 尖峰率: 12~40%
